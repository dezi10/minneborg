# Åpen Kildekode: Struktur og Beskrivelse av Nettsiden

Dette prosjektet er en portefølje/nettside for Dejan Zivanovic, med åpen og profesjonell kode. Her finner du en oversikt over alle filer, deres formål og hvordan de er strukturert. Koden er tydelig kommentert og organisert for at andre utviklere enkelt skal kunne videreutvikle eller inspisere løsningen.

## Innhold i `/files`-mappen

### HTML-filer

- **index.html**
  - Hovedside for porteføljen. Inneholder navigasjon, tilgjengelighetsfunksjoner og oversikt over innholdet.
- **arbeid.html**
  - Viser arbeidserfaring, prosjekter og relevante jobberfaringer.
- **om.html**
  - Presenterer informasjon om personen bak porteføljen, bakgrunn, utdanning og interesser.
- **kontakt.html**
  - Inneholder kontaktskjema og kontaktinformasjon. Bruker EmailJS for å sende e-post direkte fra nettsiden.
- **verktoy.html**
  - Presenterer ulike verktøy og ressurser relatert til cybersikkerhet og teknologi.
- **artikkel-1.html**
  - Fagartikkel med akademisk struktur og referanser.
- **artikkel-2.html**
  - Fagartikkel om maskinvare, ytelse og flaskehalser.
- **artikkel-3.html**
  - Fagartikkel om OSI-modellen og feilsøking i nettverk.

Alle HTML-filer har en kommentar øverst som beskriver filens formål. De benytter felles CSS og JavaScript for konsistent utseende og funksjonalitet.

### CSS-filer

- **styles.css**
  - Inneholder all stilsetting for nettsiden: layout, farger, typografi, responsivitet og tema (lys/mørk modus). Koden er strukturert med variabler og kommentarer for enkel videreutvikling.

### JavaScript-filer

- **script.js**
  - Inneholder all JavaScript for nettsiden: tilgjengelighetsfunksjoner (skriftstørrelse, lesemodus, nattmodus), språkvalg, og annen interaktivitet. Koden er modulær og kommentert for oversiktlighet.

### Bilder og SVG

- **/images/**
  - Her legges alle bilder som brukes på nettsiden.
- **/svg/**
  - Her legges alle SVG-filer, for eksempel logoer eller ikoner brukt i HTML.

## Formål og Kodekvalitet

Målet er å ha en åpen, profesjonell og lettforståelig kodebase. Alle filer er kommentert og beskrevet, slik at arbeidsgiver eller andre utviklere kan bruke inspect-funksjonen i nettleseren og enkelt forstå hvordan nettsiden er bygget opp. Dette gir full åpenhet og gjør det enkelt å videreutvikle løsningen.
